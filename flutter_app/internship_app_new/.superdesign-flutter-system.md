# StageConnect Flutter Mobile - Design System

## Product Context
**App Name**: StageConnect Mobile (Student App)
**Platform**: Flutter / iOS & Android
**Target Users**: Students searching for internships
**Brand**: Professional, modern, trustworthy internship platform

## Design Philosophy
- **Professional**: Clean, modern UI that builds trust
- **Student-friendly**: Easy to navigate, clear information hierarchy
- **Action-oriented**: Quick access to search, apply, track applications
- **Mobile-first**: Optimized for touch, thumb-friendly navigation
- **Consistent**: Matches StageConnect web brand identity

## Accessibility

### Color Contrast
- Primary blue on white: ✅ 8.59:1 (AAA)
- White on primary blue: ✅ 11.63:1 (AAA)
- Amber on white: ✅ 4.58:1 (AA)
- All text meets WCAG AA minimum

### Text Sizes
- Minimum body text: 14px
- Comfortable body text: 16px
- Sufficient line height: 1.4-1.6

### Touch Targets
- Minimum: 48x48 dp
- Spacing: 8dp minimum between targets

## Platform Guidelines

### iOS
- Use Cupertino widgets where appropriate (CupertinoNavigationBar, CupertinoButton for secondary actions)
- Respect iOS safe areas
- Use iOS-style modals (slide from bottom)

### Android
- Use Material Design 3 components
- Ripple effects on interactive elements
- FAB for primary actions
- Use Material transitions

### Adaptive Design
- Detect platform and adapt navigation style
- Use platform-appropriate haptics
- Respect platform text scaling
- Follow platform conventions (back button, swipe gestures)

## Best Practices

1. **Consistency**: Use exact hex colors, spacing values, and radii throughout
2. **Hierarchy**: Clear visual hierarchy with size, weight, and color
3. **Whitespace**: Generous spacing for mobile readability
4. **Touch-friendly**: All interactive elements 48x48 minimum
5. **Feedback**: Visual feedback for all interactions (ripple, scale, color change)
6. **Loading**: Show loading states for all async operations
7. **Errors**: Clear error messages with recovery actions
8. **Empty states**: Helpful messages with CTAs for empty screens
9. **Images**: Rounded corners, proper aspect ratios, loading placeholders
10. **Performance**: Smooth 60fps animations, optimized images, lazy loading

- Centered content, vertical layout
- Logo + app name at top
- Form fields with icons
- Primary CTA button (amber)
- Secondary actions as text buttons
- "Forgot password" link at bottom

### Dashboard / Home
- App bar with profile avatar
- Welcome message + user name
- Quick stats cards (3-4 cards in grid)
- Recent/featured offers list
- Bottom navigation

### Offer List
- Search bar at top
- Filter chips below search
- Scrollable list of offer cards
- Each card: title, company, location, apply button
- Pull to refresh

### Offer Detail
- Hero image or company logo
- Title, company name
- Location, duration, date posted
- Description sections (expandable)
- Requirements list
- Sticky "Apply" FAB at bottom

### Profile
- Header with avatar, name, school
- Info cards (contact, education)
- CV section with download option
- Edit profile button
- Settings / logout

### Applications
- Tab bar: All / Pending / Accepted / Rejected
- List of application cards
- Each card: offer title, company, status badge, date
- Tap to see details

This design system ensures the StageConnect mobile app is professional, consistent with the web brand, and optimized for student users on mobile devices.
